# Defect Reports - User Registration & Verification

| Defect ID | Summary | Steps to Reproduce | Expected Result | Actual Result | Environment | Severity | Priority | Evidence |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| DEF-001 | Email verification link uses GET request which is not supported by the backend | 1. Register a new user using the registration flow.<br>2. Open the verification email/logs.<br>3. Click the generated verification URL link: `http://localhost:5001/api/v1/auth/verify-email?token=...`. | User email is successfully verified, or the user is redirected to a success/landing page. | Browser gets an HTTP 404 error showing `Cannot GET /api/v1/auth/verify-email`. | Local Backend Server (Node.js/Express) | Medium | Medium | Express router in `authRoutes.js` only exposes `router.post('/verify-email')`. There is no `GET` handler registered for the path. |
| DEF-002 | Email duplicate check and database unique constraint are case-sensitive | 1. Register an account with email `jane@example.com`.<br>2. Attempt to register another account with email `JANE@example.com` (uppercase casing). | Registration is rejected with a `409 DUPLICATE_EMAIL` conflict error. | Registration succeeds, creating a duplicate active/pending record for the same logical email address. | Local SQLite Database | Low | Low | Column `email` in the `User` table does not declare `COLLATE NOCASE`, and `findByEmail` queries the database case-sensitively. Email Joi schema does not normalize inputs. |
| DEF-003 | Empty verification token input is ignored without feedback | 1. Complete SignUp to open the "Verify Your Email" dialog.<br>2. Leave the "Verification Token" input field blank.<br>3. Click the "Verify" button. | The UI displays an input validation error (e.g. "Verification token is required"). | The click is silently ignored with no visual error or status feedback. | Flutter Client (Web/Desktop) | Low | Low | `auth_screens.dart` line 452 returns immediately when `token.isEmpty` without showing a SnackBar or validation state message. |
