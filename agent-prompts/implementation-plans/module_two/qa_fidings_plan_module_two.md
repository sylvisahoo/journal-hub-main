# Implementation Plan - Fixing Module 2 Authentication & Session Management QA Findings

We will resolve all reported defects (**DEF-004 through DEF-009**) in Module 2 across both the backend and frontend, ensuring complete KPI compliance.

## User Review Required

> [!IMPORTANT]
> - **Draft API Implementation**: We will implement the backend endpoints for draft management (`POST /api/v1/drafts` and `GET /api/v1/drafts/:draftId`) and connect the Flutter client's auto-save to them, combined with SharedPreferences local caching for reliable offline preservation.
> - **Removal of "123456" Hardcoding**: We will remove the client-side `'123456'` code validation interceptor in the password reset flow. The real 6-digit verification code generated by the backend and sent/logged will be passed directly to the reset API.

## Verification Plan

### Automated Tests
- Run backend test suites: `npm run test`
- Run frontend widget and unit tests: `flutter test`

### Manual Verification
1. Verify token expiration: Set backend access token expiration temporarily to a very low duration (e.g. `1m`), log in, wait, and verify redirection to `/login` occurs on the next request.
2. Verify password reset: Request code, read code from terminal logs, enter the real code on the reset screen, verify password updates successfully.
3. Verify mobile logout: Shrink screen viewport to mobile size, navigate to settings, click logout, and verify redirect to login page.
4. Verify draft recovery: Start writing an entry, wait for auto-save, close the app / refresh, and verify the app displays a recovery prompt.
